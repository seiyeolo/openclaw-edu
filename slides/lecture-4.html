Chapter 02
비유로 개념 이해하기
어려운 기술 용어 없이, 일상 속 비유로 먼저 느껴봐요.
🐟
AI는 원래 금붕어예요
금붕어는 기억력이 3초라는 말 들어보셨죠? AI도 마찬가지예요.
대화창을 닫는 순간, 오늘 나눈 모든 대화가 완전히 사라져요.
내일 새 대화를 시작하면 AI는 처음 만난 사람처럼 행동해요.
문제: "지난번에 얘기했잖아요!" → AI: "처음 듣는데요?" 이 상황이 매번 반복돼요.
📒
그런데 옆에 메모장이 있다면?
금붕어 어항 옆에 메모장을 두고, 중요한 것을 적어두면 어떨까요?
다음 날 금붕어가 메모장을 읽고 나면 — 어제 일을 기억하는 것처럼 행동할 수 있어요.

GitHub의 MEMORY.md 파일이 바로 그 메모장이에요.
해결책: GitHub 메모리 파일 = AI의 메모장. 읽으면 기억한 것처럼 행동해요.
메모리 없을 때 vs 있을 때
메모리 없는 AI
🐟
월요일: "재고 47개"
↓ 대화 종료
화요일: "재고요? 모르는데요" 😵
매번 처음부터 설명해야 함
VS
메모리 있는 AI
📒🐟
월요일: "재고 47개 → MEMORY.md 저장"
↓ 대화 종료 후 재시작
화요일: "어제 재고 47개였죠? 오늘은요?" 😊
맥락을 이어서 일함

점점 더 똑똑해짐
이 사이클이 반복될수록 AI 직원의 질이 높아져요
4가지 메모리 파일 — 각각 무엇을 담나요?
📄 MEMORY.md — 프로젝트 전체 요약 (항상 읽힘)
목적, 현재 상태, 중요 규칙, 팀 구성. AI가 매 대화 시작 시 제일 먼저 읽는 파일.
↓
📋 DECISIONS.md — 결정 이유 기록 (관련 시 읽힘)
"왜 그 결정을 했나"를 이유까지 적어두는 파일. 같은 실수를 반복하지 않게 해줌.
↓
📐 PATTERNS.md — 팀 규칙·코딩 스타일 (코드 작업 시 읽힘)
네이밍 규칙, 선호 라이브러리, 코드 리뷰 기준. AI가 팀 스타일대로 작업하게 해줌.
↓
⚠️ ERRORS.md — AI 실수 모음 (디버깅 시 읽힘)
AI가 틀렸던 사례, 사용자가 수정한 내용. 같은 실수를 두 번 하지 않도록 방지.
GitHub 저장소 폴더 구조
📁 my-project/
📁 docs/ ← AI의 메모장 서랍
📄 MEMORY.md ← 항상 읽힘. 전체 요약
📄 DECISIONS.md ← 결정 이유 창고
📄 PATTERNS.md ← 팀 규칙·코딩 스타일
📄 ERRORS.md ← AI 실수 블랙리스트
📁 src/ ← 실제 코드 (메모리 아님)
📄 README.md
팁: 폴더 이름은 docs 가 관행이지만, memory 또는 ai-context 로 해도 괜찮아요.
Chapter 04
3가지 실제 사례
소상공인, 연구생·직장인, 주부·어르신 — 각자 메모리 파일에 뭘 담으면 좋을까요?
소상공인 · 퍼티스트
🧶 클레이 공방 김사장님
단골손님 취향, 인기 색상, 예약 규칙을 저장해두면 AI가 먼저 알려줘요. "이번 달 베이지 많이 나갔으니 재고 주문하세요!"
# MEMORY.md
- 단골: 박민정 → 파스텔톤
- 인기색: 베이지, 민트
- 예약: 72시간 전 마감

# ERRORS.md
- ❌ 환불 48시간 이후 처리
→ ✅ 무조건 48시간 이내
연구생 · 직장인
🔬 논문 쓰는 이연구 씨
실험 결과, 교수님 피드백, 참고문헌 스타일을 저장하면 AI가 같은 실험을 두 번 제안하지 않아요.
# DECISIONS.md
- 실험A: 2025-03 결과 불충분
- 교수님: APA 스타일 고집
- 3장 결론: 미확정

# MEMORY.md
- 제출: 6월 15일 마감
- 현재: 2장 작성 중
주부 · 어르신
👵 요리 좋아하는 박할머니
가족 알레르기, 좋아하는 음식, 단골 재료를 저장하면 AI가 "손자 땅콩 알레르기 있으니 이 레시피 빼세요"라고 먼저 알려줘요.
# MEMORY.md
- 손자: 땅콩 알레르기 ⚠️
- 남편: 매운 것 못 드심
- 단골재료: 두부, 시금치

# PATTERNS.md
- 요리법: 간단하게 설명
- 분량: 4인 기준으로
💡 세 사례의 공통점
모두 "무엇을" 이 아니라 "왜, 어떻게"를 함께 저장해요.
"카카오페이 사용" 보다 "카카오페이 선택 이유: 고객 대부분이 카카오 사용자라서" 처럼 쓸 때 AI가 진짜 판단력을 갖게 됩니다.
