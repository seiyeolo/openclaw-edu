Chapter 01
왜 AI 직원을 역할별로 나누나요?
하나의 AI에게 모든 걸 맡기면 어떤 일이 생기는지 먼저 알아봐요.
⚠️
AI 하나에 모든 걸 맡기면?
검색도 하고, 요약도 하고, 메일도 쓰고, 발송까지 하나의 AI에게 시키면 — 컨텍스트가 뒤엉키고, 판단이 흐려지고, 실수가 많아져요.
긴 작업일수록 AI가 앞에서 한 말을 잊어버리고 엉뚱한 답을 해요.
비유: 주방장이 요리도 하고 배달도 하고 계산도 하면? 요리는 타고, 배달은 늦고, 계산은 틀려요. 역할을 나눠야 잘 돌아가요.
💡
해결책 — 단일 책임 원칙 (Single Responsibility)
소프트웨어 개발의 황금 원칙을 AI에도 적용해요.
하나의 AI는 하나의 역할만 담당한다.
생각하는 역할과 실행하는 역할을 분리하면 — 각자 전문 영역에 집중해서 훨씬 좋은 결과를 내요.
핵심: "생각"과 "행동"을 분리하는 것이 에이전트 설계의 첫 번째 원칙이에요.

Chapter 02
JARVIS와 EVE를 소개합니다
두 AI 직원의 성격과 담당 업무를 자세히 알아봐요.
🍳 식당 비유로 이해하기
JARVIS = 배달부·주방보조 — 주방장이 시키는 대로 재료를 가져오고, 음식을 나르고, 설거지를 해요. 판단 없이 정확하게 실행해요.

EVE = 주방장·기획자 — 손님이 무엇을 원하는지 파악하고, 메뉴를 정하고, 어떤 순서로 요리할지 결정해요. 결과를 보고 손님에게 설명도 해요.
핵심: 주방장이 배달까지 하면 요리가 엉망이 돼요. 역할을 나눠야 잘 돌아가요!
⚡ JARVIS
실행형 에이전트 · Execution Agent
시키는 걸 정확하게, 빠르게 처리해요. 판단을 최소화하고 도구 실행에 집중해요. 짧고 구조화된 결과물을 반환해요.
담당 작업
웹 검색 캘린더 API 이메일 발송 파일 처리 DB 쿼리 문자 발송
입력: EVE의 지시
출력: JSON / 도구 실행 결과
판단: 최소화
🧠 EVE
추론형 에이전트 · Reasoning Agent
무엇을 해야 할지 판단하고, 어떻게 전달할지 결정해요. 긴 컨텍스트를 유지하며 사용자 의도를 해석해요.
담당 작업
요청 분석 계획 수립 우선순위 결정 결과 요약 사용자 응답
입력: 사용자 요청
출력: 자연어 응답
판단: 깊고 풍부하게
항목⚡ JARVIS🧠 EVE
핵심 역할도구 실행, API 호출의도 파악, 계획 수립
출력 스타일짧고 구조적 (JSON)길고 자연스러운 문장
판단 수준최소 — 그냥 실행깊음 — 맥락 전체 고려
컨텍스트짧게 유지길게 유지
비유배달부·주방보조주방장·기획자

Chapter 04
3가지 실제 사례
소상공인, 연구생·직장인, 주부·어르신 각각의 상황에서 JARVIS와 EVE가 어떻게 일하는지 살펴봐요.
🏪 사례 1 — 퍼티스트 꽃집 사장님
"이번 주 주문 정리하고 단골 손님한테 감사 메시지 보내줘"
🧠 EVE가 하는 일
의도:단골 관리 + 감사 메시지 발송 필요
계획:① 주문 조회 → ② 단골 추출 → ③ 메시지 초안 → ④ 발송
최종:메시지 내용 자연스럽게 작성 후 전달
⚡ JARVIS가 하는 일
실행:구글 시트에서 이번 주 주문 데이터 조회
실행:3회 이상 구매 단골 고객 목록 추출
실행:Solapi(문자 API)로 감사 메시지 발송
✓ 결과: "단골 고객 8분께 감사 메시지를 발송했어요. 이번 주 주문은 총 34건이었어요!"
👩‍💻 사례 2 — 논문 검토 중인 대학원생
"이 주제로 최근 3년 논문 찾아서 요약해줘"
🧠 EVE가 하는 일
의도:최신 연구 동향 파악 필요
계획:검색 키워드 설계 → 논문 선별 기준 결정
최종:논문별 핵심 내용 요약 정리 및 비교
⚡ JARVIS가 하는 일
실행:Google Scholar 웹 검색 API 호출
실행:논문 링크·초록 텍스트 추출
실행:결과 데이터 EVE에게 전달
✓ 결과: "2022~2024년 관련 논문 12편을 찾았어요. 주요 트렌드는 3가지로..."
👵 사례 3 — 병원 예약을 도와달라는 어머니
"정형외과 가까운 데 알아봐줘, 다음 주 화요일 오전에"
🧠 EVE가 하는 일
의도:병원 검색 + 예약 가능 여부 확인
계획:위치 기반 검색 → 운영시간 확인 → 안내
최종:쉬운 말로 병원 정보 안내, 예약 링크 제공
⚡ JARVIS가 하는 일
실행:네이버 지도 API로 근처 정형외과 검색
실행:화요일 오전 운영시간 데이터 수집
실행:예약 가능 병원 목록 + 전화번호 반환
✓ 결과: "화요일 오전에 진료하는 정형외과 2곳 찾았어요. 가장 가까운 곳은 도보 5분이에요!"
Chapter 05
AI 직원 설계하기 — 단계별 실습
실제로 JARVIS와 EVE를 어떻게 설계하는지 5단계로 따라해봐요.
에이전트 시스템 프롬프트 구조 예시
⚡ JARVIS 시스템 프롬프트
역할: 실행 전문 AI (JARVIS)
원칙:
- 판단 없이 지시받은 대로 실행
- 결과는 JSON으로 간결하게
- 실패 시 에러 코드 반환
- 스스로 추가 작업 금지

도구: 구글시트, 텔레그램,
        Solapi, 웹검색
🧠 EVE 시스템 프롬프트
역할: 추론 전문 AI (EVE)
원칙:
- 사용자 의도를 깊이 파악
- 작업 순서와 우선순위 결정
- JARVIS에게 명확하게 지시
- 결과를 자연스러운 한국어로

스타일: 친근하고 존댓말
💡 시스템 프롬프트가 AI 직원의 "채용 계약서"예요. 역할과 원칙을 명확히 쓸수록 AI가 더 잘 일해요.
