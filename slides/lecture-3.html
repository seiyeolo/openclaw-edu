Chapter 01
오늘 배울 것 — 한 줄 요약
복잡한 기술 없이, 이 한 줄만 기억하면 돼요.
AI에게 무엇을 · 누구에게 · 어떤 형식으로 전달하면
원하는 결과를 바로 얻을 수 있다.
이 세 가지 중 하나라도 빠지면 AI가 빈칸을 알아서 채웁니다. 그 빈칸이 내 기대와 다를 때 "AI가 이상하게 답한다"는 느낌이 들어요.
😓
이렇게 시키면 안 돼요
막연하고 짧은 지시 → AI가 "가장 평범한" 답을 내놓아요.

예: "이메일 써줘"
예: "잘 써줘"
예: "마케팅 전략 알려줘"
😊
이렇게 시키면 잘 돼요
구체적인 정보 4가지 → AI가 딱 맞는 결과물을 만들어요.

맥락 + 역할 + 형식 + 범위

이 네 가지가 프롬프트의 재료예요.
Chapter 02
비유로 개념 이해하기
초등학생도 이해하는 비유로 먼저 감을 잡아봐요.
🍽️
식당에서 주문하는 것과 똑같아요
❌ 못 시키는 주문
"밥 줘요"
→ 뭘 갖다 줄지 모르는 상황
✅ 잘 시키는 주문
"된장찌개 하나,
뜨겁게, 국물 많이,
밥 따로 주세요"
→ 정확히 원하는 대로!
AI도 마찬가지예요. 구체적으로 말할수록 원하는 결과가 나옵니다. 막연하게 시키면 AI가 "보통"으로 대답해요.
핵심: AI는 독심술사가 아니에요. 머릿속에 있는 것을 꺼내서 입력창에 넣어야 해요.
잘 시키는 것 vs 못 시키는 것 — 실제 비교
못 시키는 방법
"이메일 써줘"
잘 시키는 방법
"신규 고객에게 보낼 환영 이메일을 써줘. 우리 서비스는 SaaS 회계 툴이고, 고객은 중소기업 대표야. 친근한 톤으로 3문단 이내."
못 시키는 방법
"이 코드 고쳐줘"
잘 시키는 방법
"시니어 백엔드 개발자처럼 이 코드를 리뷰해줘. 성능 문제와 보안 취약점을 중심으로, 항목별 불릿으로 정리해줘."
못 시키는 방법
"마케팅 전략 알려줘"
잘 시키는 방법
"SNS 마케팅 전략 3가지만 알려줘. 각 항목은 제목, 실행 방법, 예상 효과 순으로. 인스타그램 중심으로, 예산 100만원 이하."
Chapter 03
도해로 구조 파악하기
좋은 프롬프트의 4가지 재료 — 한눈에 보기
좋은 프롬프트 = 4가지 재료
재료 1 — 맥락
누가 · 왜 · 어떤 상황
맥락이 없으면 AI는 가장 평범한 방향으로 해석해요.
"우리 카페", "단골 고객 대상"처럼 상황을 알려주세요.
"홍대 떡볶이 포장마차,
신메뉴 출시 상황"
재료 2 — 역할
전문가처럼 · 선생님처럼
역할을 주면 AI가 그 분야의 언어와 우선순위로 답해요.
"변호사처럼", "초등학생에게 설명하듯"처럼요.
"시니어 마케터처럼
리뷰해줘"
재료 3 — 형식
표 · 불릿 · 3줄 요약
원하는 출력 형태를 말하면 불필요한 수정 왕복이 없어져요.
표, 번호 목록, 문단, 3줄 요약 등을 미리 정하세요.
"불릿 3개,
각 항목 1줄씩"
재료 4 — 범위
이것만 · 이건 빼고
빼고 싶은 것을 말하는 게 더 효과적이에요.
"가격 언급 제외", "UX 흐름만"처럼 경계를 그어주세요.
"전문용어 빼고,
이 부분만 봐줘"
↓
완성된 프롬프트
AI가 바로 이해하는 지시문
↓
원하는 결과물 출력
처음 시도부터 바로 쓸 수 있는 결과
재료 4개 = 맥락 + 역할 + 형식 + 범위
하나라도 빠지면 AI가 알아서 채워요 — 내 기대와 다르게!
프롬프트 품질의 차이 — 바깥에서 안으로
정보가 많아질수록 결과 품질이 올라가요
🟢 최상 — 맥락 + 역할 + 형식 + 범위 + 예시
AI가 마치 오랜 동료처럼 정확히 이해하고 즉시 쓸 수 있는 결과물을 만들어요
↓ 조금 빠짐
🔵 좋음 — 맥락 + 형식 + 범위
한두 번 수정하면 원하는 결과가 나와요
↓ 더 빠짐
🟡 보통 — 맥락만 있음
방향은 맞지만 형식이나 범위 조정이 필요해요
↓ 거의 없음
🔴 최하 — 한 단어, 동사 하나
AI가 빈칸을 마음대로 채워요. 내 기대와 다른 결과가 나올 확률이 매우 높아요
Chapter 04
3가지 실제 사례
소상공인 · 연구생·직장인 · 주부·어르신 — 각자의 상황에 맞는 프롬프트 예시예요.
🏪 소상공인 — 떡볶이 가게 사장님
인스타그램 홍보글을 써야 하는데 뭘 말해야 할지 모를 때.

✅ 이렇게 쓰세요
"우리 가게(홍대 떡볶이 골목, 2평 포장마차) 신메뉴 치즈떡볶이 홍보 인스타 글 써줘. 20~30대 직장인 대상, 이모지 포함, 3줄 이내."
맥락(홍대 포장마차) + 대상(20~30대) + 형식(3줄, 이모지) = 완성!
🎓 연구생·직장인 — 논문 다듬기
논문 초록을 학술지에 투고할 수 있게 다듬어야 할 때.

✅ 이렇게 쓰세요
"아래 논문 초록을 학술지 투고용으로 다듬어줘. 영어 원어민 수준, 250단어 이내, 수동태 줄이고 명확한 기여점 강조."
역할(원어민) + 범위(수동태, 기여점) + 형식(250단어) = 완성!
🏠 주부·어르신 — 저녁 메뉴 고르기
냉장고에 있는 재료로 오늘 저녁을 뚝딱 만들고 싶을 때.

✅ 이렇게 쓰세요
"냉장고에 달걀, 두부, 파, 간장이 있어. 20분 안에 만들 수 있는 저녁 반찬 2가지만 알려줘. 레시피는 단계별로 짧게."
맥락(재료) + 범위(20분, 2가지) + 형식(단계별) = 완성!
📌 세 사례의 공통점
공통된 실수
"이메일 써줘" "레시피 알려줘" 처럼 동사 하나만 씀
공통된 성공 패턴
상황 + 대상 + 제한 조건 + 원하는 형태를 함께 씀
기억하세요: 길게 쓰는 게 중요한 게 아니라, 맥락·역할·형식·범위 중 어떤 게 빠졌는지 확인하는 게 중요해요.

원칙한 줄 정의비유중요도
맥락누가, 왜, 어떤 상황인지 알려주기식당에서 상황 설명하기핵심
역할AI에게 어떤 전문가 역할을 줄지직원에게 담당 업무 주기중요
형식표, 불릿, 몇 줄인지 미리 지정보고서 양식 주기핵심
범위포함할 것·빼야 할 것 명시주문할 때 "빼주세요" 말하기중요
예시 제공원하는 톤·스타일을 예시로 보여주기참고할 샘플 주기중요
수정 요청"더 짧게", "톤 바꿔줘"로 이어가기편집 과정으로 대화하기참고
프롬프트는 초안처음부터 완벽할 필요 없다첫 보고서가 완성본이 아닌 것처럼참고
